# Build Setup Documentation

## Overview

This project uses a custom build setup with **two separate HTML files**:
- `index.template.html` - **Source template** (DO NOT COMMIT CHANGES TO THIS FILE)
- `index.html` - **Built output** (Generated automatically during build)

## File Structure

```
project-root/
├── index.template.html    # Source template (references /src/main.jsx)
├── index.html             # Built output (references /assets/index.js)
├── src/
│   ├── main.jsx          # Entry point
│   ├── App.jsx           # Main app component
│   └── ...
├── assets/
│   ├── index.js          # Built JavaScript
│   └── index.css         # Built CSS
└── vite.config.js        # Build configuration
```

## How It Works

### 1. Source Template (`index.template.html`)

This is the **source file** that Vite uses as input during the build process. It contains:

```html
<script type="module" src="/src/main.jsx"></script>
```

**Important:** This file is automatically restored after each build. Any manual changes will be overwritten.

### 2. Built Output (`index.html`)

This is the **production file** generated by the build process. It contains:

```html
<script type="module" crossorigin src="/assets/index.js"></script>
<link rel="stylesheet" crossorigin href="/assets/index.css">
```

This file is used in production and should be committed to git.

### 3. Build Process

When you run `npm run build`, the following happens:

1. **Pre-build**: Deletes old `index.html` to ensure clean build
2. **Build**: Vite reads `index.template.html` and compiles all source files
3. **Output**: Creates `index.template.html` with built asset references
4. **Post-build**: 
   - Renames built `index.template.html` → `index.html`
   - Restores source `index.template.html` with `/src/main.jsx` reference

## Usage

### Development

```bash
# Start development server
npm run dev
```

The dev server uses `index.template.html` and serves source files directly.

### Production Build

```bash
# Build for production
npm run build
```

This will:
- Compile all source files from `src/`
- Generate optimized `assets/index.js` and `assets/index.css`
- Create production-ready `index.html`
- Restore source `index.template.html`

### Making Changes

1. **Edit source files** in `src/` directory (e.g., `src/App.jsx`)
2. **Run build**: `npm run build`
3. **Verify changes**: Check that `assets/index.js` contains your changes
4. **Commit**: Commit both source files and built files

### Troubleshooting

#### Changes not appearing in build output

**Solution 1: Clear Vite cache**
```bash
rm -rf node_modules/.vite
npm run build
```

**Solution 2: Verify source template**
```bash
# Check that index.template.html has the correct reference
grep "src/main.jsx" index.template.html
```

**Solution 3: Force clean build**
```bash
rm -rf node_modules/.vite
rm -f index.html assets/index.js assets/index.css
npm run build
```

#### Build fails with "Could not resolve entry module"

This means `index.template.html` is missing or corrupted. The build process will automatically restore it, but you can manually restore it:

```bash
# The source template is embedded in vite.config.js
# Just run the build again and it will be restored
npm run build
```

## Git Workflow

### Files to Commit

✅ **Always commit:**
- `index.template.html` (source template)
- `index.html` (built output)
- `assets/index.js` (built JavaScript)
- `assets/index.css` (built CSS)
- All source files in `src/`

❌ **Never commit:**
- `node_modules/`
- `.vite/` cache

### Recommended Workflow

```bash
# 1. Make changes to source files
vim src/App.jsx

# 2. Build for production
npm run build

# 3. Verify changes
grep "your-change" assets/index.js

# 4. Commit everything
git add .
git commit -m "Your changes"
git push
```

## Configuration

The build setup is configured in `vite.config.js`:

```javascript
export default defineConfig({
    plugins: [react(), manageIndexPlugin()],
    build: {
        outDir: '',                    // Build to root directory
        assetsDir: 'assets',           // Assets go to /assets
        base: './',                    // Relative paths
        rollupOptions: {
            input: 'index.template.html',  // Use template as input
            output: {
                entryFileNames: 'assets/index.js',
                assetFileNames: 'assets/index.css',
            }
        }
    }
})
```

The `manageIndexPlugin()` handles:
- Deleting old `index.html` before build
- Renaming built template to `index.html`
- Restoring source template

## Why This Setup?

This setup solves the problem where:
1. You need to commit built files to git (for deployment)
2. You need to maintain a source template for development
3. Vite needs to know which file to use as input

By separating the source template and built output, we ensure:
- ✅ Clean builds every time
- ✅ No confusion between source and built files
- ✅ Easy deployment (just commit and push)
- ✅ Source changes always reflected in build output

## Quick Reference

| File | Purpose | Contains | Commit? |
|------|---------|----------|---------|
| `index.template.html` | Source template | `/src/main.jsx` | ✅ Yes |
| `index.html` | Built output | `/assets/index.js` | ✅ Yes |
| `src/App.jsx` | Source code | Your code | ✅ Yes |
| `assets/index.js` | Built JavaScript | Compiled code | ✅ Yes |
| `node_modules/.vite` | Build cache | Temp files | ❌ No |

## Support

If you encounter any issues with the build process:

1. Clear the cache: `rm -rf node_modules/.vite`
2. Delete built files: `rm -f index.html assets/index.js assets/index.css`
3. Rebuild: `npm run build`
4. If still failing, check `vite.config.js` for the source template content
